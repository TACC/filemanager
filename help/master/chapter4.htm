Managing Resources

 

Introduction

This chapter covers how to identify and manage the resources accessible through the XFM. For a full listing of XSEDE Resources, consult the XSEDE User Portal System Monitor page.

 

Managing Resources

When the XFM starts, the left panel will automatically open up to your home directory on your local file system. The right panel will display the Resource List of systems to which you have access.  This list includes both XSEDE systems on which you have accounts as well as systems that you manually add yourself.  The XSEDE systems you see correspond to the systems listed under “System Accounts” in the Accounts and Usage portlet.  These cannot be removed from the Resource List.  Systems you have manually added using the add/remove buttons at the bottom of the Resource List panel can be edited and removed at any time.

 

Selecting Resources

If you look above the directory listing on both the left and right panels, you will see a combo drop down box with the current resource name displayed.  This combo box, like the one you saw when you first opened the XFM, contains a list of every resource to which you have access in the XSEDE.  Selecting a resource from the list will take you to your home directory on the selected resource.  Selecting resources from the combo boxes provide a way for you to navigate from resource to resource in the XSEDE.

 

Adding Resources

At the bottom of the Resource List panel you will see two buttons.  Clicking the button with a plus sign will open an Add New Resource dialog allowing you to specify the name, hostname, and connection properties for an arbitrary resource.  Two requirements apply to any system added in this dialog. First, it must be accessible via GridFTP.  Second, it must accept your XSEDE credentials.  If either of these two conditions are not met, you will not be able to access this system through the XFM.

 

Editing Resource Properties

To edit the properties of any resource, right click it in the Resource List and select Edit.  This will bring up the Edit <Name> dialog.  In this dialog you can edit the connection properties of any resource as well as tune the file transfer parameters such as parallelism, striping, and buffer size.  While you may change the name, category, and hostname of any resource that you manually added, these properties cannot be changed for the XSEDE resources automatically populated for you by the XFM.

 

Removing Resources

At the bottom of the Resource List panel you will see two buttons.  Clicking the button with a minus sign will remove any manually added resource from the Resource List.  Auto-populated XSEDE resources cannot be removed from the Resource List.

 

XSEDE Share

XSEDE Share is a new service providing all users with 2GB of secure, personal space for collaborating with other XSEDE users and the world.  XSEDE Share allows you to share files and folders with colleagues on your own terms. No setup is needed to use this new service.  You will see XSEDE Share appear automatically towards the bottom of your resource list when you start the file manager.

Through the file manager, you can browse and manage data in XSEDE Share like any other system.  The first time you access your XSEDE Share account, you will notice two convenient &qt;smart folders&qt; by default.  The first folder is your shared folder. This shows you all the files and folders that other people have shared with you.  You can browse and edit the content in this folder just like any other folder you have access to.

The second folder is your public folder.  This folder represents your public space. The data in this folder is publicly available to anyone on the web.  The file manager makes it easy for you to publish files to your public folder.  Simply browse to the file you want to publish on any system in your resource list and drag that file into your public folder.  Your public folder is an easy way to post data for others to download, publish documents or images that you’ll need for a meeting, or simply save something that you need to access often.

To get a unique URL for any file in your Share account, public or private, simply right click on the file and select &qt;Get Public/Shared URL&qt;.  For files you have shared with others, but not made public, the link can be accessed by any of the people sharing that document by authenticating using their XSEDE User Portal username and password.  For files you have not shared, you alone can access the URL using your XSEDE User Portal username and password.  Public files require not authentication.

When you want to start working with others, you’ll need to give them permission to your files and/or folders.  You can do this through the file info panel. Instead of UNIX permissions, XSEDE Share items have share permissions listed in their information panel.  This is basically just a list of other users and the permissions you’ve given them. Here you can add, remove, or change permissions on a file or folder for other users.  Once you’ve shared an item with someone, they will get an email letting them know that you shared that item with them and they will immediately have access to it from their shared folder.

 

Removing Resources

The file manager also allows you to move data into and out of your personal Amazon S3 account.  To enable S3 access, go to the File->Preferences dialog and click on the S3 tab.  Click the &qt;Enable S3 access&qt; checkbox and enter your S3 key and secret.  Close the dialog. Now your S3 account will show up as another resource in your resource list.  You can now browse and manage data in your S3 account like any of your other resources.
