Browsing Files

 

Introduction

 

Viewing and manipulating the contents of multiple file systems is the primary purpose of the XFM. This section covers how to view, manipulate and manage your files across the XSEDE.

 

Browsing Files

When you select a resource from the resource combo box, the XFM will establish a session with the selected resource and display a listing of your home directory.  The listing will show the name, size, date modified, and permission attributes of each file or folder.  The parent folder will also be included in the listing as the first entry with a name of  “..”.

In order to navigate down through the file system, double click on a folder to list its contents.  To navigate back up through the file system, either click on the parent directory entry or select a parent folder from the path combo box immediately above the file listing panel.  The path combo box contains an entry for every folder in the absolute path of the current folder.  Selecting any entry will take you to the folder.

Several shortcut buttons have also been provided to help you navigate quickly through the file system.

 

Go To

Displays an input dialog box where you can specify the path (absolute or relative) of the folder you wish to open.

 

Refresh

The XFM automatically refreshing panel listing periodically and as a result of an event completing.  However, to manually refresh a folder listing, the refresh button is provided.

 

Home

Takes you to your home directory.

 

The XFM also supports single-panel and detached viewing.  This means that you have the flexibility of viewing a single file browsing panel at a time, or opening up an instance of the XFM in a separate window rather than working directly in your browser window.  When opening an instance of the XFM in detached view, there are two options available.  The first is to open up a fresh instance of the XFM.  This will be the same thing you see when you first open the XFM page.  The other is to open up a detached instance of the XFM with the current listings of the embedded XFM.  Both options are available via the applet menu bar’s File option.

 

Info Dialog

 

The XFM provides many familiar features common to your native file manager.  One such feature is the Info Dialog. This dialog shows detailed information about the selected file or folder. By default, the name, size, path and document type icon are shown. The remaining information listed in the dialog is given below.

 

Type

This is the authentication method used to access the file or directory. Valid values are either GridFTP, XSEDE_SHARE, AMAZON S3, or FILE.

 

System

This is the name of the system on which the file or directory resides.

 

Path

This is the absolute path of the file or directory on the resource.

 

Permissions

This expandable field shows the Unix permissions on a file. For remote systems, file permissions are editable through this interface.  Changes are effective immediately.

 

Transfers

 

File uploads, download, and third party transfers are all supported in the XFM.  To quickly download a file to your local $HOME directory, either double click on the file or right click on the file and select “Download” from the pop up menu. All other file movements can be performed by dragging files from one listing to the other.

 

Directory transfers are supported.  Currently the XFM creates the remote folder then transfers every file in the subdirectory tree.  In the case a folder or file of the same name already exists, you will be prompted to overwrite or rename the transferring file/folder.

 

File transfers are all tracked and monitored from within the history table at the bottom of the XFM window.  (If the history table is not visible, you may make it visible by selecting View->Show Transfers from the top application menu.) When you request a file transfer, the request is queued up at the bottom of the table.  Once the transfer starts, the progress bar updates as the transfer takes place.  Transfer speed, predicted finish time, parallelism, etc are all displayed in real time as the transfer proceeds.  In addition, the transfer information is persisted in the XFM middleware for future reference.

 

As with file browsing, several shortcut buttons have been provided to help you quickly manage your files.

 

Rename

Rename the selected folder or file.  An input dialog box will allow you to specify the new name.

 

New Folder

Create a new folder in the currently displayed folder listing.

 

Delete

Delete the current file or folder.  You will be prompted to verify this action every time.

 

For information on how to receive notifications of file transfer start and completion information, see Chapter 7 on Notifications.

 

Predictions

 

Often times, it’s helpful to get an idea of how long a file transfer would take in advance.  We have added a Transfer Time Prediction feature to the XFM based on information obtained from PSC’s Speedpage.  This feature is meant to provide rough estimates about the best case 3rd party transfer time between XSEDE resources.  This feature is currently in beta release and should be provide, at best, a rough estimate.

 

Bookmarks

 

To help you locate important file and folders on different machines, bookmark support has been added to the XFM.  The Bookmark panel is available by clicking on the bookmark button (the one with the book on it) in the file listing panel.

 

Any file or folder can be bookmarked.  You can enter them in manually via the add/remove bookmark buttons in the Bookmark panel, or add them automatically from the right click menu on a file or folder. 

 

Bookmarks are set relative to a particular machine and will be stored from one use of the file manager to another.

 

Search

 

A Search panel has been added to allow you to look for files on a remote system.  The Search panel is available by clicking on the search button (the one with the magnifying glass on it) in the file listing panel. 

 

To avoid excessively long searches, two parameters are available in the Preferences dialog: Maximum Search Depth and Maximum Search Results.  The former specifies how many levels in depth from the currently listed folder the XFM will search.  The latter specifies the maximum number of search results that the XFM will return.  If either  of these limits are met in the course of the search, the search will immediately terminate.

 

Using Your Environmental Variables

 

A An Environment variable panel has been added to allow you to access your environment variables on XSEDE resources.   The Environment panel is available by clicking on the environment button (the one with the recycling icon on it) in the file listing panel. By default, (and for clarity) you will only see your environmental variables referring to system paths in this listing.
