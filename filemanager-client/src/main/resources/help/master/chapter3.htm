Authenticating

 

Introduction

This chapter covers the authentication and authorization techniques used by the XFM. For more information on Grid security, please consult the Globus project.

 

Authenticating

The XFM leverages X.509 security within the Globus Grid Security Infrastructure (GSI) to authenticate users to access XSEDE resources. Specifically, the XFM uses the GridFTP protocol to establish secure connections to remote resources. By default the XFM uses your XSEDE credential from the main MyProxy server throughout the life of your session. This credential is retrieved when you log into the portal and passed to the XFM at startup.

If the portal credential is not active, the XFM, or simply expires, you will be prompted to re-enter your XSEDE User Portal username and password to obtain another credential. This alleviates the problem of you having to log out and log back into the portal.

When running the file manager as a desktop application, you will be prompted to enter your username and password at login to retrieve your credential from the XSEDE MyProxy service. If this credential expires, you can refresh it at any time by using the File->Login dialog.
